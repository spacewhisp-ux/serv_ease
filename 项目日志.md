# Serv Ease 项目日志

## 2026-04-23 后端功能增强

### 1. 修复用户回复已关闭工单的问题

**问题描述：**
用户可以回复已关闭的工单，并且工单会自动重新打开。这与业务需求不符。

**修复内容：**
- 在 `TicketsService.replyTicket` 方法中添加状态检查
- 如果工单状态为 CLOSED，抛出 BadRequestException
- 移除了自动重新打开工单的逻辑

**修改文件：**
- [server/src/tickets/tickets.service.ts](server/src/tickets/tickets.service.ts)

**验证结果：**
- ✅ 用户无法回复已关闭的工单
- ✅ 返回错误信息："Closed tickets cannot be replied to"

- 在 `AdminService.listTickets` 方法中实现搜索逻辑
- 使用 PostgreSQL 的不区分大小写搜索（mode: 'insensitive'）

### 4. 实现工单操作历史功能

**功能描述：**
记录工单的所有操作历史，包括创建、状态变更、分配、回复、关闭等操作。

**数据库变更：**
- 创建 `ticket_histories` 表
- 添加 `TicketHistoryAction` 枚举（CREATED, STATUS_CHANGED, ASSIGNED, REASSIGNED, PRIORITY_CHANGED, CATEGORY_CHANGED, REPLIED, CLOSED, REOPENED）
- 添加 `TicketHistoryActorRole` 枚举（USER, AGENT, ADMIN, SYSTEM）

**实现内容：**
- 创建 `TicketHistoryService` 服务
- 在以下操作中记录历史：
  - 用户创建工单（CREATED）
  - 用户回复工单（REPLIED）
  - 用户关闭工单（CLOSED）
  - 管理员分配工单（ASSIGNED/REASSIGNED）
  - 管理员回复工单（REPLIED，仅公开回复）
  - 管理员更新工单状态（STATUS_CHANGED）
- 添加 API 接口 `GET /v1/admin/tickets/:id/history`

**新增文件：**
- [server/src/tickets/ticket-history.service.ts](server/src/tickets/ticket-history.service.ts)
- [server/prisma/migrations/20260423_add_ticket_history/migration.sql](server/prisma/migrations/20260423_add_ticket_history/migration.sql)

**修改文件：**
- [server/prisma/schema.prisma](server/prisma/schema.prisma)
- [server/src/tickets/tickets.module.ts](server/src/tickets/tickets.module.ts)
