## Project Overview

This repository is currently a near-stock Flutter application scaffold with one Dart entrypoint and one widget test. Most generated platform code lives in `android/`, `ios/`, `macos/`, `linux/`, `windows/`, and `web/`; application logic currently lives entirely in `lib/main.dart`.

The only substantial product-specific artifact today is `DESIGN.md`, which documents the intended visual system. If you are implementing UI, use `DESIGN.md` as the source of truth for theme, spacing, typography, border radius, and the monochrome Expo-inspired look.

## Tooling

- Flutter SDK must satisfy `pubspec.yaml`'s SDK constraint: Dart `^3.12.0-327.1.beta`
- The local environment currently matches Flutter beta / Dart beta (`flutter --version` reports Flutter `3.44.0-0.1.pre`, Dart `3.12.0`)
- Linting is configured through `analysis_options.yaml`, which includes `package:flutter_lints/flutter.yaml`

## Architecture

### App structure

- `lib/main.dart` is the full app entrypoint today
- `main()` calls `runApp(const MyApp())`
- `MyApp` builds the root `MaterialApp`
- `MyHomePage` is the default scaffold screen generated by Flutter
- State is local widget state in `_MyHomePageState`; there is no separate state-management, domain, or data-access layer yet

Because the app is still scaffold-level, new product work will likely require introducing structure rather than extending existing architecture. Keep that in mind before assuming there are established feature/module boundaries elsewhere in the repo.

### Testing structure

- `test/widget_test.dart` is the only existing test
- It boots `MyApp` directly and verifies the default counter flow
- Use it as the baseline example for future widget tests until a richer test harness exists

### Platform code

- Native/platform runner code is generated in the standard Flutter directories (`android/`, `ios/`, `macos/`, `linux/`, `windows/`, `web/`)
- Prefer keeping app behavior in `lib/` unless the task explicitly requires platform-specific integration

## Design Context

`DESIGN.md` is the important non-generated document in this repository. The key implementation constraints from it are:

- Bright cool-white background with white elevated surfaces
- Monochrome visual hierarchy with black headlines and blue-gray secondary text
- Rounded-to-pill geometry across buttons, containers, and previews
- Large display typography with tight letter-spacing
- Spacious, gallery-like vertical layout rhythm

If a UI change conflicts with the default Flutter Material starter styles, prefer the design direction in `DESIGN.md` over the scaffold defaults.
